# Отчет по разработке библиотеки Smart Tax Tools

## 1. Обзор решения
Проект представляет собой специализированную библиотеку на Python для выполнения точных финансовых расчетов: вычисления налоговых обязательств и чистого дохода.

## 2. Логика работы и архитектура
В данной библиотеке в основе лежат два алгоритма, реализованные в модуле `calculations.py`:

* **Алгоритм расчета налога (`calculate_tax`):** Принимает доход и ставку, возвращает абсолютную сумму налога.
* **Алгоритм чистого дохода (`calculate_net_income`):** Комплексная функция, агрегирующая доход, налоговую ставку и фиксированные операционные сборы (fees).

### Принципы By-design
В коде реализован механизм защиты от логических ошибок бизнеса:
1.  **Ограничение отрицательного баланса:** Результат функции `calculate_net_income` всегда пропускается через фильтр `max(result, 0)`. Это исключает ситуации, при которых налоги и сборы приводят к отрицательному доходу.
2.  **Строгая типизация:** Использование `Union types` (напр. `int | float`) совместно с проверкой `mypy` гарантирует, что в функции не попадут некорректные данные из внешних источников.

## 3. Автоматизация и управление (Makefile)
Все этапы жизненного цикла проекта автоматизированы через `Makefile`.

### Доступные таргеты:
* **Установка и среда:**
    * `make install`: Создает виртуальное окружение `.venv` и устанавливает необходимые зависимости (`flake8`, `mypy`, `pytest`, `build`, `twine`).
* **Контроль качества (QA):** 
    * `make lint`: Проверка исходного кода и тестов на соответствие стандарту PEP 8.
    * `make typecheck`: Статический анализ типов.
    * `make test`: Запуск модульных тестов.
    * `make all`: Последовательное выполнение всех проверок качества.
    * `make format`: Автоматическое форматирование исходного кода инструментом **Black** для соблюдения единого стиля.
    * `make check`: Составной таргет для быстрого запуска линтера, проверки типов и зависимостей.
    * `make check-deps`: Анализ импортов на соответствие `requirements.txt` через **pip-check-reqs**. Позволяет выявить неиспользуемые или забытые пакеты.
* **Сборка и публикация:**
    * `make build`: Очистка проекта от временных файлов и сборка дистрибутивов (`.whl` и `.tar.gz`).
    * `make publish`: Загрузка собранных пакетов в индекс TestPyPI.
    * `make clean`: Удаление кэшей (`__pycache__`, `.pytest_cache`, `.mypy_cache`) и артефактов сборки.
